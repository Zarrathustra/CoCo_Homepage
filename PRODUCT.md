# Product

## Register

brand

## Users

CoCo Project serves structural biology laboratories, especially cryo-EM researchers, graduate students, PIs, and scientific software users who need a clear, approachable entry point for related cryo-EM analysis tools.

## Product Purpose

The homepage should help visitors quickly understand that CoCo is the unified GUI interface for the CoCo Project. CoCo wraps and connects three submodules: CryoSieve, CryoPROS, and CTFA. The primary conversion goal is downloading CoCo; the secondary goal is understanding what CoCo does and how the four software pieces relate.

## Brand Personality

Gentle, professional, academic. The interface should feel credible for a research software project while remaining calm, readable, and welcoming to lab users who want practical clarity rather than marketing spectacle.

## Anti-references

Avoid generic research-project template pages, exaggerated SaaS landing-page styling, overly flashy dark tech aesthetics, and dense jargon walls that make the software relationship harder to understand.

## Design Principles

1. Explain the system relationship first: CoCo is the GUI hub, and CryoSieve, CryoPROS, and CTFA are the wrapped analysis modules.
2. Make download confidence high: the primary action should be easy to find, visually calm, and supported by enough context to feel trustworthy.
3. Prefer academic clarity over sales language: use precise, readable copy that respects scientific users.
4. Keep the experience gentle and composed: visual polish should support comprehension, not compete with the content.
5. Show modular capability without fragmentation: the homepage should make the four tools feel like one coherent project.

## Accessibility & Inclusion

Target WCAG AA by default. Maintain strong contrast, color-blind-safe module distinctions, keyboard-accessible navigation, readable typography, and reduced-motion support for any animations.
